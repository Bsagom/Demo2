📂 프로젝트 설계안: G-Insight1. 프로젝트 개요 (Overview)프로젝트명: G-Insight (G-Market Insight Assistant)목표: 방대한 주문 데이터를 분석하여 MD(상품 기획자)에게 **'돈이 되는 프로모션'**과 **'충성 고객'**을 직관적으로 보여주는 비즈니스 대시보드 구축.사용 데이터: Kaggle Olist E-commerce Dataset (주문, 상품, 고객, 결제, 리뷰 데이터 포함)2. 타겟 유저 (User Persona)Primary User: 카테고리 담당 MD (Merchandiser)Pain Point: "지난번 20% 할인 쿠폰을 뿌렸는데, 매출은 올랐지만 실제 이익(마진)이 남았는지 모르겠다.", "이번 신상품을 누구에게 문자로 홍보해야 할지 모르겠다."Secondary User: 마케팅 팀장Pain Point: "우리 플랫폼의 고객 재구매율이 떨어지고 있는 시점을 알고 싶다."3. 핵심 기능 명세 (Core Features)지마켓 실무에서 가장 중요하게 보는 3가지 영역을 기능으로 구현합니다.A. 프로모션 수익성 분석기 (Promotion ROI Analyzer)기능: 특정 기간(이벤트 기간)의 GMV(총 거래액) 대비 순수익 시뮬레이션.Input: 분석 기간 선택, 예상 마진율 입력 (데이터셋에 원가가 없으므로 시뮬레이션 값 입력).Output:Revenue (매출): 판매가 총합.Profit (영업이익): 매출 - (매출 * 할인율) - 배송비 - (매출 * 예상원가율)Insight: "할인율 15%일 때 주문수는 폭증하지만, 영업이익은 -2%로 적자 전환됩니다. 적정 할인율은 10%입니다." 라는 텍스트 자동 생성.B. 고객 세분화 엔진 (RFM Segmentation)기능: 고객의 구매 패턴을 점수화하여 등급 자동 분류.Logic (SQL 활용):Recency (최근 구매일): 최근 구매일이 오늘로부터 며칠 지났는가?Frequency (구매 빈도): 기간 내 몇 번 주문했는가?Monetary (구매 금액): 총 얼마를 썼는가?Output:VIP 고객군: "최근 30일 내 구매, 총 50만 원 이상 구매" → 신규 런칭 프로모션 타겟.이탈 위험군: "과거 VIP였으나 최근 90일간 구매 없음" → 컴백 쿠폰 발송 타겟.C. 카테고리 교차 판매 분석 (Basket Analysis)기능: 특정 카테고리 구매자가 함께 구매한 상품군 분석.Output: 히트맵(Heatmap) 시각화.예: "가구(Furniture)" 카테고리 구매자의 30%가 "인테리어 소품(Decoration)"을 1주일 내에 구매함. → "가구 상세페이지 하단에 인테리어 소품 추천 배너 삽입 제안"4. 데이터 구조 및 스키마 (ERD 설계)이 프로젝트의 핵심은 **"데이터를 어떻게 조인(Join)해서 지표를 뽑는가"**입니다. 아래 4개 테이블을 중심으로 Star Schema를 구성합니다.Orders (Fact Table): 주문 ID, 고객 ID, 주문 상태, 구매 시각 (order_purchase_timestamp)Order_Items (Fact Table): 주문 ID, 상품 ID, 판매가(price), 배송비(freight_value)Products (Dimension): 상품 ID, 카테고리명 (product_category_name)Customers (Dimension): 고객 ID, 고객 고유 ID(customer_unique_id), 지역 정보[핵심 SQL 포인트]단순 customer_id가 아닌 customer_unique_id를 사용하여 유저 레벨 통합 분석을 수행해야 함 (지마켓 면접관이 유심히 볼 포인트).5. 기술 스택 (Tech Stack)Database: MySQL (AWS RDS 프리티어 사용 권장) 또는 로컬 SQLite.Query: SQL (Window Function, CTE, Group by 필수 사용).Language: Python 3.9+.Web Framework: Streamlit (빠르고 직관적인 대시보드 제작).Visualization: Plotly (인터랙티브 차트), Matplotlib.6. 구현 로드맵 (2주 완성 계획)단계기간내용산출물1주차 (Data)Day 1-3Kaggle 데이터 다운로드 및 SQL DB 적재 (전처리 포함)ERD, DB 파일Day 4-5SQL 쿼리 작성 (RFM 분석, 일별/월별 매출 집계 쿼리)SQL 스크립트 모음 (.sql)2주차 (App)Day 6-7Python Pandas로 데이터 로드 및 로직 검증Jupyter NotebookDay 8-10Streamlit 대시보드 UI 구현 (차트 연동)웹 어플리케이션Day 11-12인사이트 도출 및 포트폴리오 문서 작성PDF 보고서, GitHub 리포지토리7. 지마켓 합격을 위한 '한 끝' (Killer Point)면접관에게 어필하기 위해, 대시보드 상단에 단순히 숫자만 띄우지 말고 "Action Item" 영역을 만드세요.(예시 화면)현재 상황: 20대 여성 고객의 이탈률이 전월 대비 5% 증가했습니다.Action Item: "뷰티 카테고리 무료 배송 쿠폰" 발송 시 예상 회복 매출은 약 2,000만 원입니다.
